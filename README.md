**Ola Data Analysis Dashboard**
This project analyzes data from Ola ride bookings to pull out some insights on how bookings happen, why some get cancelled, and what the revenue looks like overall. 

OBJECTIVES
To check total bookings and see the success rate for rides.
Cancellation rate calculation
Understanding customers and drivers behaviours.
Trends in demand for rides.

Tools
Power BI for visuals. 
SQL helped with querying the data,
Excel for some initial cleaning.

INSIGHTS
Key things I tracked included total bookings, the ones that got cancelled, and the rate for cancellations. Successful rides and revenue were the big metrics too. Revenue part gets a bit tricky, it varies.

In the dashboard, I added previews, like screenshots of the charts. They show the trends clearly, I hope. One insight stood out, high cancellations in some areas. Peak hours for bookings are around evenings, it seems. Revenue changes over time, not steady at all.

LEARNINGS
From this, I learned about cleaning data and transforming it right. Writing DAX for measures like cancellation rate and KPIs took practice. Dashboard design, making it tell a story, that part is not totally straightforward. Some areas still feel underdeveloped, like deeper driver behavior.
